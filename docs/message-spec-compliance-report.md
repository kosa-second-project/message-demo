# 메시징 시스템 기능 명세 반영 확인 보고서

작업 브랜치: `codex/message-spec-ui-audit`

## 점검 기준

- 사용자가 붙여준 회의 내용/노션 명세를 기준으로 섹션별 반영 여부를 확인했다.
- 기존 디자인 시스템의 카드, 배지, 버튼, 테이블, 차트 톤을 유지했다.
- 보류 또는 명세 충돌이 있는 부분은 화면을 방해하지 않는 작은 `Info` 핀으로 설명했다.

## 1. 계정 및 권한

- 로그인/로그아웃: 반영.
- 계열사 정보 기반 로그인: 반영. 로그인 화면에서 계열사 선택 후 권한 안내 표시.
- 관리자 권한 구분: 데모 수준 반영. 계열사 선택에 따른 권한 안내만 표시.
- 세션 관리: 데모 앱 상태 기반 반영.
- 메뉴 및 기능 접근 제어: 반영. 로그인 후 사이드바 메뉴 접근.
- 개인정보 조회/수정 권한 관리: 부분 반영. 회원 상세 조회 중심이며 권한별 세분화는 데모 범위 밖.
- 관리자 작업 이력 관리: 미반영. 명세에서 취소선 처리되어 각 기능 내 기록으로 대체.

## 2. 대시보드 - 보류

- 오늘 발송 현황: 반영.
- 발송 성공률/실패 현황: 반영.
- 활성 회원 현황: 반영.
- 월 누적 발송 현황: 반영.
- 스마트 라우팅 절감 현황: 반영.
- 실시간 발송 큐 상태: 반영.
- 최근 발송 추이: 반영.
- 채널별 발송 비중: 반영.
- 최근 전송 기록 요약: 반영.
- 주요 업무 바로가기: 반영.
- 보류 상태 설명: 반영. 주요 업무 바로가기 영역에 작은 핀으로 표시.

## 3. 메시지 발송

### 3-1. 수신자 선택

- 타겟 기반 발송 대상 선택: 반영. 명세의 태그 용어는 정책에 맞춰 타겟으로 표시.
- 카테고리 중복 고객 1회 발송: 반영 의도. UI는 중복 제거된 예상 대상 수로 표시.
- 다중 타겟 AND/OR 선택: 반영. `모두 포함`, `하나라도 포함` 토글 제공.
- 검색된 회원 전체 선택: 반영.
- 개별 회원 포함/제외: 반영. 포함 대상/제외 대상 박스 복구.
- 회원 직접 추가: 반영.
- 수신 동의 및 수신거부 반영: 반영. 수신 가능한 채널이 없는 회원은 수신자 목록에서 제외.

### 3-2. 채널 선택 - 보류

- 동의 안 된 채널 선택 제한: 부분 반영. 회원별 수신 가능 채널은 표시, 채널별 완전 차단 로직은 데모 범위.
- 비용 기반 권장 플랫폼 자동 선택: 반영. 기본값은 스마트 라우팅/최저비용 추천.
- 추가 선택/해제: 반영.
- 보류 상태 설명: 반영. 채널 선택 헤더에 작은 핀 표시.

### 3-3. 메시지 작성 - 보류

- 템플릿 기반 메시지 작성: 반영.
- 직접 메시지 작성: 반영.
- 메시지 변수 삽입 및 치환: 반영.
- 스마트폰 형태 미리보기: 반영. 기본/카카오/리치 3종 토글.
- SMS/LMS 길이 판별: 반영.
- 채널별 발송 조건 검증: AI 검사 단계에서 반영.
- 발송 채널 선택, 스마트 라우팅, 대체 발송: 반영.
- 예상 발송 비용/절감액: 반영. 대체 라우팅 실패 비용 제외 안내 포함.
- 즉시/예약 발송, 최종 확인: 반영.
- 보류 상태 설명: 반영. 메시지 미리보기 영역에 작은 핀 표시.

## 4. AI 발송 지원

- 타겟 추천: 반영.
- 마케팅 문구 추천: 반영. AI 자동 추천 모드에서 최종 문구까지 제안.
- 템플릿 저장: 반영. 메시지 작성 단계에 템플릿 저장 버튼 제공.
- 맞춤법/오타 검사: 반영.
- 광고 표기 검사: 반영.
- 민감 표현 검사: 반영.
- 채널 길이 적합성 검사: 반영.
- 브랜드 톤앤매너 검사: 반영.
- AI 검사 리포트: 반영.

## 5. 템플릿 관리

- 템플릿 목록/검색: 반영.
- 타겟/카테고리/채널 필터: 반영.
- 카테고리 필수 선택: 반영.
- 타겟 다중선택/수정/추가/삭제: 회원 타겟 관리와 템플릿 필터 수준 반영.
- 상세 정보: 반영. 카테고리, 타겟, 채널 표시.
- 핸드폰형 메시지 UI 3종: 반영.
- 사용 횟수, 클릭률, 수신거부율: 반영.
- 템플릿 등록/수정/삭제: 반영.
- 템플릿 AI 검사 결과: 미반영. 취소선 처리되어 제거.
- 템플릿 승인 상태 관리/카톡 심사 상태: 미반영. 취소선 처리되어 UI와 목업 데이터에서 제거.

## 6. 전송 기록

- 기간/상태/채널/계열사 필터: 반영.
- 템플릿명 검색: 반영.
- 발송일시, 계열사, 템플릿, 채널, 대상, 발송, 성공, 실패, 비용 절감, 1차 성공률, 상태: 반영.
- 엑셀 내보내기: 반영.
- 상세의 절감액 보류: 반영. 상세에서는 절감액을 보류 메모로 처리.
- 메시지 정보, 최종 도달률, 대체 발송 단계별 결과, 실패 원인 분류, 후속 조치: 반영.

## 7. 회원 관리

- 회원 목록: 반영.
- 이름, 전화번호, 타겟, 채널별 동의 여부, 가입일: 반영.
- 회원 파일 내보내기: 반영.
- 회원 파일 추가: 미반영. 명세에서 제거 요청된 항목이라 UI에서 제외.
- 유형 일반/휴면/신규만 표시: 반영. VIP 제거.
- 수신거부자 목록: 반영.
- 080 수신거부 반영: 반영.
- 수신거부자 파일 내보내기: 반영.
- 수신거부자 상세에서 회원 상세 컴포넌트 재사용: 반영.
- 회원이 받은 문자 내역: 반영.

## 8. 회원/템플릿 태그 관리 정책

- 태그 -> 타겟 용어 변경: 반영. 화면 노출 용어는 타겟으로 통일.
- 타겟 유형 고정: 반영 의도. 기본 타겟 그룹을 고정하고 사용자 추가 타겟은 별도 그룹으로 분리.

## 9. 통계

- 발송 현황 fallback 건수/전환율: 반영.
- 시간대별 발송 분포 -> fallback 도넛: 반영. 도넛 차트로 교체.
- 일간/주간/월간 드롭다운: 반영.
- 1차/2차/3차 성공률 + 건수: 반영.
- hover 시 채널별 건수: 반영.
- 채널 분석의 채널별 추이 및 전체 기간 선택: 반영.
- 비용/라우팅 분석 용어 실제 비용 vs 최대 비용: 반영.
- 월별 절감액 추이: 반영.
- Failover 전환 건수, 계열사별 비용 점유 제거: 반영.
- 발송 현황/비용 라우팅 위치 교환: 반영.
- 회원 분석의 회원 용어, 일반/신규/휴면만 표시: 반영.
- 회원 유형별 오픈율 & 성공률, 상세 지표 제거: 반영.
- 신규 회원 가입자 수 그래프: 반영.
- 성과 분석 평균 오픈율 제거: 반영.
- 채널 드롭다운: 반영.
- 요일별 클릭률, 시간별 클릭률: 반영.
- 월별 성과 지표 추이에서 오픈율 제거: 반영.
- 템플릿별 성과 Top 5, 클릭률/구매전환율, AI 템플릿 표시: 반영.
- PDF/Excel 다운로드: 반영.

## 배포/품질 확인

- 루트 `index.html`은 Vite 소스 진입점으로 복구했다.
- GitHub Pages Actions는 `main` 배포용 소스 빌드 흐름으로 복구했다.
- 작업 브랜치는 `/docs` 정적 산출물을 갱신하여 브랜치 `/docs` 배포 설정에 대응했다.
- `vite build --outDir docs` 성공.
