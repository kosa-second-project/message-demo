# 현대퓨처넷 메시징 시스템 데모

현대퓨처넷 메시징 운영자를 위한 웹 프론트엔드 데모입니다. 실제 백엔드 연동 전 회의와 기능 검토를 위해 만든 화면 중심 프로토타입이며, 로그인부터 메시지 발송, 템플릿 관리, 전송 기록, 회원 관리, 통계까지 주요 운영 흐름을 포함합니다.

## 실행 방법

```bash
pnpm install
pnpm dev
```

개발 서버는 LAN 접속 확인을 위해 `0.0.0.0`으로 열립니다.

- 로컬: `http://127.0.0.1:5174`
- 같은 네트워크: `http://192.168.2.247:5174`

데모 로그인은 아무 ID/비밀번호 입력 후 진행할 수 있습니다.

## 주요 기능

### 1. 로그인

- 운영자 로그인 화면
- 데모 입력 검증
- 로그인 후 운영 대시보드 진입
- 실제 구현 시 Spring Security, 암호화, 마스킹 적용을 전제로 한 화면 구조

### 2. 대시보드

- 오늘 발송 건수
- 발송 성공률
- 활성 회원 수
- 이번 달 누적 발송
- 일별 발송 추이 그래프
- 채널별 발송 비중
- 최근 전송 기록
- 빠른 작업 바로가기
- AI 마케팅 인사이트 카드

### 3. 메시지 발송

단계형 발송 플로우로 구성되어 있습니다.

1. 수신자 선택
2. 메시지 작성
3. 채널 선택
4. 검토 및 발송

포함 기능:

- 태그 기반 대상 선택
- 수백 개 이상 태그를 고려한 검색형 태그 UI
- 유사 태그 추천
- 회원 개별 검색
- 회원 부분 포함/제외 선택
- 수동 수신자 추가
- 포함 대상/제외 대상 목록 관리
- 대량 템플릿을 고려한 검색형 템플릿 목록
- 템플릿 선택 즉시 메시지 미리보기
- 변수 치환 예시 미리보기
- 메시지 직접 작성
- SMS, LMS, RCS, 카카오 알림톡, 카카오 친구톡 채널 선택
- AI 자동 추천으로 대상, 템플릿, 채널, 문구를 한 번에 구성
- 소형 LLM 여러 개를 비동기로 실행하는 AI 검사 UI
- 오타, 광고 표기, 민감 표현, 개인정보, 채널 길이, 발송 피로도 검사
- AI 검사 리포트
- 즉시 발송/예약 발송 선택

### 4. 템플릿 관리

- 마케팅 템플릿 목록
- 템플릿 검색
- 태그/카테고리 기반 필터
- 템플릿 추가, 수정, 삭제
- 행 클릭 시 상세 팝업
- 템플릿별 채널, 사용 횟수, 수정일, 태그 확인
- 페이지네이션
- AI 도구 사용을 고려한 관리 화면 구조

### 5. 전송 기록

- 발송 이력 목록
- 캠페인/템플릿명, 채널, 대상, 발송 수, 성공/실패 수, 상태 확인
- 행 hover 인터랙션
- 행 선택 시 상세 팝업
- 성공률, 실패 사유, 대상 조건, AI 검사 결과 등 데이터 확인용 구조

### 6. 회원 관리

- 회원 목록 조회
- 회원명, 전화번호, 회원 유형, 태그, 수신 동의 상태 확인
- SMS, LMS, RCS, 카카오 동의 상태 관리 구조
- 회원 상세 팝업
- 전체 변경/부분 변경을 고려한 운영 UI
- Excel, CSV, JSON 파일 업로드 UI
- 다중 태그 보유 회원 구조
- 페이지네이션

### 7. 통계

통계는 4개 페이지로 분리되어 있습니다.

- 통계 개요
  - 발송 성과 요약
  - 주요 KPI
  - 기간별 성과 그래프
- 채널 통계
  - 채널별 발송량
  - 성공률/클릭률/반응률 비교
  - 채널별 비용 효율 판단용 지표
- 회원 통계
  - 회원 태그별 반응
  - 세그먼트별 성과
  - 동의 채널별 회원 분포
- 성과 분석
  - 캠페인별 성과 비교
  - 템플릿/문구 성과
  - 재발송 또는 개선 대상 파악

## 화면 구성

- `src/app/App.tsx`: 전체 화면과 mock 데이터가 들어 있는 메인 앱 파일
- `src/main.tsx`: React 진입점
- `src/styles/theme.css`: 디자인 토큰
- `src/styles/index.css`: 스타일 엔트리
- `src/app/components/ui`: shadcn 기반 UI 컴포넌트

## 기술 스택

- React
- TypeScript
- Vite
- Tailwind CSS
- Recharts
- Lucide React

백엔드는 아직 연결하지 않은 프론트엔드 데모입니다. 실제 프로젝트 구현 시 Spring Boot, Thymeleaf, Oracle, Spring Security와 연결하는 것을 전제로 합니다.

## 참고

- 현재 데이터는 모두 mock 데이터입니다.
- 대량 회원, 대량 태그, 대량 템플릿 운영을 고려해 검색/필터/페이지네이션 중심 UI로 설계했습니다.
- 실제 발송, AI 검사, 파일 업로드, 회원 수정은 아직 API 연동 전 화면 데모입니다.
