# 메시징 시스템 기능 명세

## 문서 기준

이 문서는 현재 구현된 메시징 시스템 화면을 기준으로 주요 기능 구성을 정리한 문서다.  
세부 정책, API 명세, DB 설계, 화면 와이어프레임은 별도 문서에서 다룬다.

## 1. 계정 및 권한

관리자 사용자가 시스템에 접근하고, 역할에 따라 사용할 수 있는 기능을 제한하는 영역이다.

주요 기능:

- 관리자 로그인/로그아웃
- 세션 관리
- 관리자 권한 구분
- 메뉴 및 기능 접근 제어
- 개인정보 조회/수정 권한 관리
- 관리자 작업 이력 관리

## 2. 대시보드

메시지 발송 운영 현황을 빠르게 확인하는 첫 화면이다.

주요 기능:

- 오늘 발송 현황
- 발송 성공률/실패 현황
- 활성 회원 현황
- 월 누적 발송 현황
- 스마트 라우팅 절감 현황
- 실시간 발송 큐 상태
- 최근 발송 추이
- 채널별 발송 비중
- 최근 전송 기록 요약
- 주요 업무 바로가기

## 3. 메시지 발송

운영자가 발송 대상을 정하고, 메시지를 작성한 뒤 채널과 발송 방식을 선택해 발송 요청을 생성하는 핵심 기능이다.

주요 기능:

- 태그 기반 발송 대상 선택
- 전체 회원 대상 발송
- 개별 회원 포함/제외
- 수동 수신자 추가
- 수신 동의 및 수신거부 반영
- 템플릿 기반 메시지 작성
- 직접 메시지 작성
- 메시지 변수 삽입 및 치환
- 스마트폰 형태 메시지 미리보기
- SMS/LMS 길이 판별
- 채널별 발송 조건 검증
- SMS, LMS, 카카오 알림톡, 카카오 친구톡, RCS 발송 채널 선택
- 스마트 라우팅 선택
- 대체 발송 설정
- 예상 발송 비용 계산
- 예상 절감액 계산
- 즉시 발송
- 예약 발송
- 발송 전 최종 확인

## 4. AI 발송 지원

메시지 발송 과정에서 AI를 활용해 발송안을 추천하고 메시지 품질을 검사하는 기능이다.

주요 기능:

- 발송 대상 추천
- 템플릿 추천
- 채널 추천
- 마케팅 문구 추천
- 추천 결과 발송 설정 반영
- 맞춤법/오타 검사
- 광고 표기 검사
- 민감 표현 검사
- 개인정보 노출 검사
- 채널 길이 적합성 검사
- 발송 피로도 검사
- AI 검사 리포트 제공

## 5. 템플릿 관리

반복 발송에 사용하는 메시지 템플릿을 관리하고, 템플릿별 성과와 승인 상태를 확인하는 기능이다.

주요 기능:

- 템플릿 목록
- 템플릿 검색
- 태그/카테고리/채널 필터
- 템플릿 상세 정보
- 템플릿 등록
- 템플릿 수정
- 템플릿 삭제
- 템플릿 사용 범위 관리
- 템플릿 승인 상태 관리
- 반려 사유 관리
- 승인 필요 채널의 템플릿 사용 제한
- 템플릿 사용 횟수
- 템플릿 오픈율/클릭률/수신거부율
- 템플릿 AI 검사 결과

## 6. 전송 기록

발송 요청 이후의 처리 결과를 확인하고, 실패 원인과 대체 발송 흐름을 추적하는 기능이다.

주요 기능:

- 전송 기록 목록
- 기간/상태/채널/계열사 필터
- 템플릿명 및 채널명 검색
- 전송 기록 상세 정보
- 요청/성공/실패 건수
- 발송 비용 및 절감액
- 1차 성공률
- 최종 도달률
- 대체 발송 단계별 결과
- 실패 원인 분류
- 실패 원인별 후속 조치

## 7. 회원 관리

발송 대상이 되는 회원 정보와 채널별 수신 동의, 수신거부, 대량 업로드를 관리하는 기능이다.

주요 기능:

- 회원 목록
- 회원 검색
- 회원 유형 필터
- 회원 상세 정보
- 회원 태그
- SMS 수신 동의
- 카카오 수신 동의
- RCS 수신 동의
- 수신 동의 변경
- 수신거부자 목록
- 080 수신거부 반영
- 회원 파일 업로드
- 업로드 데이터 검증
- 업로드 반영 전 미리보기

## 8. 통계 및 분석

발송 결과와 회원 반응을 분석해 운영 성과를 확인하는 기능이다.

주요 기능:

- 전체 발송 현황
- 월별/일별 발송량
- 채널별 발송량
- 채널별 성공률/실패율
- 채널별 비용 및 평균 단가
- 스마트 라우팅 절감 효과
- 대체 발송 전환 현황
- 계열사별 발송량 및 비용
- 회원 유형별 발송 성과
- 채널별 수신 동의 현황
- 오픈율/클릭률/전환율/수신거부율
- 템플릿별 성과 순위
- 요일별 성과
- PDF 다운로드
- Excel 다운로드

## 9. 발송 엔진

발송 요청을 실제 발송 처리 흐름으로 연결하고 결과를 수집하는 백엔드 처리 영역이다.

주요 기능:

- 발송 큐 등록
- 발송 큐 상태 관리
- 대량 발송 분할 처리
- 예약 발송 스케줄 처리
- 발송 성공/실패 결과 수집
- 대체 발송 처리
- 최종 실패 기록
- 전송 기록 반영
- 발송 비용 계산
- 스마트 라우팅 절감액 계산

## 10. 외부 연동

실제 메시지 발송과 부가 처리를 위해 외부 시스템과 연결되는 영역이다.

주요 기능:

- SMS 발송 시스템 연동
- LMS 발송 시스템 연동
- 카카오 알림톡 발송 시스템 연동
- 카카오 친구톡 발송 시스템 연동
- RCS 발송 시스템 연동
- 080 수신거부 시스템 연동
- AI 추천/검사 시스템 연동
- 파일 저장소 연동
- 통계 다운로드 파일 생성 연동

## 11. 보안 및 감사

개인정보 보호와 운영 추적성을 확보하기 위한 기능이다.

주요 기능:

- 개인정보 마스킹
- 개인정보 원문 접근 권한 제한
- 개인정보 조회 이력
- 발송 요청 로그
- 템플릿 변경 로그
- 회원 정보 변경 로그
- 파일 업로드 로그
- 다운로드 로그
- 오류 로그

## 12. 오류 및 예외 처리

사용자 입력 오류, 업무 정책 위반, 외부 시스템 실패를 안정적으로 처리하는 기능이다.

주요 기능:

- 필수 입력값 누락 처리
- 잘못된 예약 일시 처리
- 발송 대상 없음 처리
- 수신 동의자 없음 처리
- 승인되지 않은 템플릿 사용 차단
- 채널별 발송 제한 위반 차단
- 권한 없는 접근 처리
- AI 검사 실패 처리
- 파일 업로드 실패 처리
- 외부 발송 연동 실패 처리
- 통계 조회 실패 처리
- 다운로드 생성 실패 처리

## 13. 개발 완료 기준

서비스 개발 완료 시 다음 흐름이 실제 데이터 기준으로 연결되어야 한다.

완료 기준:

- 발송 대상 선정부터 발송 요청 생성까지 연결
- 발송 요청부터 전송 기록 반영까지 연결
- 전송 기록 기준 통계 집계
- 수신 동의와 수신거부 정보의 발송 대상 반영
- 템플릿 승인 상태의 발송 가능 여부 반영
- AI 검사 완료 조건의 발송 가능 여부 반영
- 권한별 기능 접근 제한
- 주요 변경 작업 감사 이력 저장
- 주요 오류 상황에 대한 사용자 안내 및 서버 로그 저장
